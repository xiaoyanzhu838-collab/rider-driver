# 12 / 22 总线电机驱动说明

日期：2026-04-17

适用对象：
- 总线电机 ID `12`
- 总线电机 ID `22`

说明：
- 这份文档基于现有硬件实测、日志分析、寄存器扫描和联调结果整理。
- 目前没有 ESP32 端官方原始控制程序，因此这里是“当前已验证可用”的驱动说明，不是官方规格书复刻。

## 1. 当前结论总览

- 这两个电机使用的是一条串口总线，协议行为与 `AX-12 / DYNAMIXEL Protocol 1.0` 高度兼容。
- 当前实测可用波特率为 `1000000`（1 Mbps）。
- 两个电机机械安装方向是镜像关系：
  - `12` 的自由运动方向是“往更小的位置值走”
  - `22` 的自由运动方向是“往更大的位置值走”
- 反方向硬推时会明显进入高负载，且能看到 `0x20` 错误，强烈指向 Protocol 1.0 的 overload 语义。

## 2. 已确认可用的寄存器

当前已经确认读写正常的关键地址如下：

| 功能 | 地址 | 长度 | 说明 |
| --- | --- | --- | --- |
| Torque Enable | `0x18` | 1 byte | `1` 上力，`0` 卸力 |
| Goal Position | `0x1E` | 2 bytes | 目标位置 |
| Goal Speed | `0x20` | 2 bytes | 目标速度 |
| Present Position | `0x24` | 2 bytes | 当前位置 |

另外还验证过以下反馈区可读：

| 功能 | 地址 | 长度 |
| --- | --- | --- |
| Present Speed | `0x26` | 2 bytes |
| Present Load | `0x28` | 2 bytes |
| Present Voltage | `0x2A` | 1 byte |
| Present Temperature | `0x2B` | 1 byte |
| Moving | `0x2E` | 1 byte |

## 3. 控制表特征

寄存器扫描结果与 AX 风格控制表高度一致，当前识别结果：

- model = `12`
- firmware = `28`
- return delay = `250`
- angle range table 读出来是 `0 .. 1023`

这说明当前位置值可以按 `0~1023` 量级去理解。

## 4. 当前已验证的机械安全范围

这是目前最重要的结论，已经在固件和 Android 调试 app 中落地。

### 电机 12

- 基准中心位约为 `599`
- 自由方向：朝更小值移动
- 当前稳定可达边界：`599 -> 495`
- 也可以理解为：
  - last good abs delta = `104`
  - first fail abs delta = `105`

### 电机 22

- 基准中心位约为 `439`
- 自由方向：朝更大值移动
- 当前稳定可达边界：`439 -> 551`
- 也可以理解为：
  - last good abs delta = `112`
  - first fail abs delta = `113`

## 5. 边界附近的行为

边界附近不是简单“突然报码停止”，而是更像机构受限后的平衡点。

### 电机 12

- 当目标打到大约 `493` 附近时，开始出现“目标到不了”
- 最终会稳定在 `496 ~ 500` 一带

### 电机 22

- 当目标打到大约 `551 ~ 552` 的失败边界附近时，开始出现“目标到不了”
- 最终会稳定在 `544 ~ 547` 一带

结论：
- 这更像机械连杆 / 负载限制导致的可达边界
- 不是简单通信错误
- 也不是单纯因为速度不够

## 6. 速度参数的理解

已测试速度值：

- `80`
- `200`
- `400`

现象：
- 改变速度会影响前期过渡过程
- 但不会明显改变最终的极限平衡位置

因此当前建议：
- 日常调试先用 `200`
- 如果只是做动作验证，`80 ~ 400` 都可以
- 不要指望通过单纯加大速度突破机械边界

## 7. 反馈值解释建议

### Present Position

- 可直接作为当前位置使用
- 单位可按 AX 风格位置值理解

### Present Speed

- 更适合按“有符号 16 位”解释，不建议直接当无符号值看
- 当前观察到：
  - `12` 在自由方向运动时，通常偏正
  - `22` 在自由方向运动时，通常偏负

### Present Load

- 可做定性参考
- 但当前这套硬件的 load 编码表现，不建议生搬 AX 教材里的位语义做强判断

## 8. 当前推荐控制流程

建议按下面的顺序驱动：

1. 使能扭矩：`Torque Enable = 1`
2. 写入目标位置：`Goal Position`
3. 写入目标速度：`Goal Speed`
4. 轮询读取当前位置：`Present Position`
5. 如需释放：`Torque Enable = 0`

如果是双电机联动，建议使用同步写：

- 起始地址：`0x1E`
- 连续长度：`4`
- 内容依次为：
  - goal position low/high
  - goal speed low/high

## 9. 当前推荐的软件安全限位

当前软件中建议固化以下限位，不要在常规控制里越界：

| ID | Home | Edge | 推荐范围 |
| --- | --- | --- | --- |
| `12` | `599` | `495` | `495 .. 599` |
| `22` | `439` | `551` | `439 .. 551` |

联动展开百分比建议这样映射：

- `12`：`home - delta`
- `22`：`home + delta`

其中：

- `12` 最大安全 delta = `104`
- `22` 最大安全 delta = `112`

## 10. 当前可直接套用的联动模型

如果定义展开百分比 `percent = 0..100`，当前推荐：

### 电机 12

```text
goal12 = 599 - round(104 * percent / 100)
```

### 电机 22

```text
goal22 = 439 + round(112 * percent / 100)
```

例如 `25%` 展开时，实测联动结果为：

- `12 -> 573`
- `22 -> 467`

## 11. 当前实用命令层建议

如果从上层应用封装动作，建议至少保留这些语义动作：

- `HOME`
  - `12 -> 599`
  - `22 -> 439`
- `EDGE`
  - `12 -> 495`
  - `22 -> 551`
- `PAIR percent speed`
  - 按当前百分比联动展开
- `SET id goal speed`
  - 对单个电机安全设位
- `TORQUE id 0|1`
  - 单独上力 / 卸力
- `RELAX`
  - 两个电机一起卸力
- `CAPTURE_HOME`
  - 把当前实际位置记为新的中心位

## 12. 当前风险与注意事项

- 当前结论来自反向分析和实测，不是官方程序定义。
- 机械边界已经比较清楚，但还不是“绝对极限图谱”，不要直接越过当前安全范围做长期运行。
- 如果要继续往极限附近试探，建议单步、小速度、短时间，并实时看位置反馈。
- 不建议在上电自启动里直接跑极限测试逻辑。
- 目前更适合把这两个电机当成“已知安全区间内可控”的总线执行器来使用。
